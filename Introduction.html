Apa Itu Cloud Computing?

Teknologi penyedia sumber daya digital (server, storage, database, network) secara virtual via internet dengan metode sewa tanpa mengelola infrastruktur fisik sendiri.

Analogi Utilitas Listrik

Anda tidak perlu membangun pembangkit listrik di rumah untuk menyalakan lampu. Cukup hubungkan ke stopkontak, bayar tagihan sesuai watt yang dikonsumsi.

1. On-Demand Self-Service

Sumber daya langsung aktif saat dibutuhkan tanpa campur tangan manusia.

2. Broad Network Access

Akses menyeluruh dari mana saja lewat koneksi internet standar.

3. Resource Pooling

Resource fisik dipool bersama lalu dialokasikan dinamis sesuai permintaan multi-tenant.

4. Rapid Elasticity

Mudah meningkatkan atau menurunkan performa CPU/RAM secara real-time.

5. Measured Service

Penggunaan resource dipantau, dikontrol, dan ditagih secara transparan (pay-per-use).

Model Layanan Cloud IaaS • PaaS • SaaS

Infrastructure as a Service

Menyediakan infrastruktur dasar IT virtual: server, storage, dan jaringan.

  • User kelola: OS, Middleware, Runtime, Data, Aplikasi
  • Provider kelola: Virtualisasi, Server Fisik, Storage, Jaringan
  • Contoh: AWS EC2, GCP Compute Engine, Azure VM

Platform as a Service

Lingkungan runtime dan pengembangan aplikasi siap pakai tanpa setup OS.

  • User kelola: Data & Aplikasi
  • Provider kelola: OS, Runtime, Middleware, Jaringan, Server
  • Contoh: AWS Elastic Beanstalk, Heroku, Google App Engine

Software as a Service

Aplikasi jadi yang diakses langsung via browser tanpa instalasi lokal.

  • User kelola: Hanya konfigurasi penggunaan personal
  • Provider kelola: Seluruh tumpukan aplikasi dan infrastruktur
  • Contoh: Google Drive, Zoom, Microsoft 365, Trello
Analogi Pizza as a Service
  • 🍕 On-Premise: Menguleni adonan, siapkan bahan, panggang di oven rumah sendiri.
  • 📦 IaaS: Beli pizza setengah matang, panggang sendiri di oven rumah.
  • 🛵 PaaS: Pizza diantar matang. Anda sediakan meja & alat makan.
  • 🏪 SaaS: Duduk manis di restoran, makan, bayar, lalu pulang.

Jenis Deployment Model Cloud Arsitektur Jaringan

Public Cloud

Infrastruktur digunakan bersama banyak organisasi (Multi-Tenant). Semua resource diatur virtual oleh pihak ketiga.

Contoh: AWS, Azure, GCP, Alibaba Cloud.

Private Cloud

Infrastruktur didedikasikan penuh untuk satu organisasi. Kontrol keamanan sangat ketat dan compliance tinggi.

Contoh: OpenStack internal BUMN, cloud privat bank.

Hybrid Cloud

Menggabungkan public dan private cloud. Data sensitif di lokal, beban kerja umum di public cloud.

Contoh: Aplikasi lokal tersinkron ke AWS Backup.

💡 Community Cloud: Berbagi infrastruktur untuk organisasi dengan regulasi/kebutuhan sama (contoh: Cloud Terintegrasi antar Kementerian/Dinas Provinsi).

Docker: Kontainer Kargo Digital Containerization

Mengapa Menggunakan Docker?

Mengatasi masalah klase "Berjalan di laptop saya, tapi crash di production". Docker mengemas program beserta pustaka pendukung secara modular.

Analogi Kapal Kontainer

Memuat karung beras, drum minyak, dan peti kayu ke kapal sangat kacau. Revolusi datang dengan Kontainer Standar yang seragam — apapun isinya, kontainer ditumpuk rapi di kapal mana pun.

Docker: 50–100+ unit

Mengapa Docker Lebih Ringan?

Container Docker berbagi kernel OS host secara langsung, mengisolasi resource pada level proses. Virtual Machine memerlukan instalasi Guest OS utuh di atas Hypervisor, memakan RAM dan storage ekstra besar.

Cloud Requirements dalam SRS Dokumen Standar

Kebutuhan Fungsional (FR)

  • Sistem harus terintegrasi dengan cloud storage gateway untuk upload dokumen laporan masyarakat.
  • Sistem harus melakukan backup database otomatis terjadwal setiap hari ke server cadangan cloud.
  • Sistem harus mengirim notifikasi push real-time menggunakan layanan cloud messaging pihak ketiga.

Kebutuhan Non-Fungsional (NFR)

  • Availability: Uptime minimal 99.9%.
  • Scalability: Auto-scaling horizontal hingga 10x beban puncak tanpa downtime.
  • Security: HTTPS (TLS 1.3) untuk transmisi, AES-256 untuk data at-rest.
  • Disaster Recovery: RTO < 4 jam, RPO < 1 jam.
  • Latency: Respons API < 200ms pada kondisi normal.

Sistem Informasi Disnaker

Merancang sistem penanganan laporan pencari kerja dengan arsitektur cloud terdistribusi:

  • Frontend & Backend: Di-package ke Docker Container untuk pengujian staging maupun cloud.
  • Database: Managed MySQL (RDS/Cloud SQL) dengan enkripsi penuh untuk data NIK & KTP.
  • Penyimpanan Gambar: Upload lampiran laporan langsung ke Object Storage S3 via API.
